12:T7bb,**TL;DR: Claude's Microsoft 365 connector moved from read-only to read-write on July 7 — email draft/send, calendar create/update, and OneDrive/SharePoint file creation are now available; admins must enable write tools for existing connections.**

The M365 connector has been search-and-read since launch: Claude could query your mailbox, read calendar, and browse SharePoint, but couldn't act. The July 7 update adds write tools.

**What's new:**
- **Email**: draft from scratch, send on your behalf, organize folders, and manage drafts — within your existing M365 permissions
- **Calendar**: create, update, and delete events; schedule meetings, set reminders, manage recurring events
- **OneDrive & SharePoint**: create and update Word documents, Excel spreadsheets, PowerPoint presentations, and raw files in your org's SharePoint environment
- **Mailbox settings**: update rules, signatures, and configurations

**What stays read-only:** Microsoft Teams — Claude can read messages and channels, but write access to Teams has not been enabled.

**Admin rollout:** Organizations that already had the connector enabled have write tools **blocked by default** until an admin explicitly enables them in the connector settings. New connections can enable or disable write tools during setup.

**Why it matters:** For engineering teams building Claude-powered internal workflows — weekly status reports drafted from project data, meetings scheduled from planning output, docs created from design decisions — the connector now closes the last step that previously required a separate M365 API integration.
