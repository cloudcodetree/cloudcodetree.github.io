A clean architectural pattern is hardening for managing context cost. MCP tool definitions are expensive: a few connected servers (e.g., GitHub + Playwright + an IDE integration) can occupy ~70%+ of a 200K context window before the agent acts, and measured tool-selection accuracy drops sharply (one test fell from 43% to under 14%) as tool count grows — the practical ceiling is ~5–7 servers. Skills, by contrast, use progressive disclosure (~30–50 tokens of metadata until invoked). The recommended shape: pin one MCP per external system you genuinely need live (GitHub, Postgres, Linear, Sentry), and push orchestration logic into thin skills.
