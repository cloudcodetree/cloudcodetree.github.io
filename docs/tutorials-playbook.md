# Tutorials playbook

How the hand-authored **Tutorials** section works and how to add a new one. This
is separate from the auto-generated AI News blog (different content, different
pipeline).

## The three layers

1. **AI News blog** (`/`, auto) — daily breadth, including a teachable curriculum post.
2. **Tutorials** (`/tutorials`, hand-authored MDX) — durable, interactive depth.
3. **Companion code repos** (`github.com/cloudcodetree/tutorial-<slug>`) — the
   actual code, built as a **step-by-step git-tag progression** so learners can
   check out or diff any stage.

A tutorial links to its companion repo; the repo's README links back to the tutorial.

## Two tutorial types

- **Verified** (the default) — locally runnable code with a companion repo. Every
  code block and every result shown is from **actually running it**. If a claimed
  result doesn't reproduce, fix the example until it does — never fabricate output.
  The `RAG from Scratch` series is all verified.
- **Anchored** — for GPU/version-sensitive topics that can't be locally tested
  (fine-tuning, serving). No companion repo and **no "tested locally" claims**.
  Teach the concept, mark code snippets as illustrative, and link an **official,
  verified-live notebook/doc** (open the URL before linking it). Use a
  `<Callout tone="warn">` to say where the code actually runs. The
  `Fine-Tuning & Serving` series is anchored.

## Conventions

- **Series + part.** Every tutorial belongs to a series. The canonical display
  title is `"<series>: <subtitle> (Part N of M)"` — `fullTitle()` in the manifest
  composes it; it's what goes in `<title>`/Open Graph so shared links read as a
  series. The UI shows a `series · Part N/M` eyebrow + the subtitle.
- **Hero cover image.** Each tutorial has a branded 16:9 cover in
  `public/tutorials/covers/<slug>.png`, generated by
  `scripts/generate-tutorial-covers.mjs` (dark gradient + accent + series/part/
  subtitle). Shown in list cards, as the article hero (`<TutorialHero>`), and as
  the per-post OG image. Committed (small fixed set), unlike the blog's CDN images.
- **Verified type only:** **one repo per tutorial**, public, named
  `tutorial-<slug>` under `cloudcodetree`; **one annotated git tag per step**
  (`step-01`, `step-02`, …; `main` = finished); README **step table with GitHub
  compare links** (`/compare/step-01...step-02`).
- `<slug>` is lowercase letters/digits/hyphens and matches across the repo name,
  the route (`/tutorials/<slug>`), and the MDX folder.

## Adding a new tutorial

### 1. Companion code repo

```bash
# scaffold a sibling repo with step-01 already committed + tagged
node scripts/new-tutorial-repo.mjs <slug> --title "Human Title" --create-remote
```

Then build the code, committing and tagging each step, and push the tags:

```bash
cd ../tutorial-<slug>
# …make step 2's changes…
git add -A && git commit -m "step 2: <what>"
git tag -a step-02 -m "Step 2 - <what it adds>"
git push origin main --tags
```

Add a row to the README's compare table per step. (Omit `--create-remote` to
scaffold locally first and create the GitHub repo later.)

### 2. Site tutorial (MDX)

First add an entry to `app/tutorials/manifest.ts` (`title` is the **subtitle**):

```ts
{
  slug: '<slug>',
  title: '<Subtitle>',           // part-specific; the series is separate
  series: '<Series Name>',       // shared across the series
  part: <n>,                     // 1-based position within the series
  excerpt: '<one-line summary>',
  date: 'MM-DD-YYYY',
  tags: ['Tutorial', '<Topic>', '…'],
  order: <global n>,             // ascending = global learning-path position
  readTime: <minutes>,
  image: '/tutorials/covers/<slug>.png',
}
```

Generate its cover (add the entry to the list in the script first):

```bash
node scripts/generate-tutorial-covers.mjs
```

Then create `app/tutorials/(article)/<slug>/page.mdx`. Use the composed title
(`"<series>: <subtitle> (Part N of M)"`) in `<title>`/OG so previews read as a
series, and add the cover as the OG image:

```mdx
export const metadata = {
  title: '<Series>: <Subtitle> (Part N of M) · Tutorials',
  description: '<one-line summary>',
  alternates: { canonical: 'https://cloudcodetree.com/tutorials/<slug>/' },
  openGraph: {
    title: '<Series>: <Subtitle> (Part N of M)',
    description: '<summary>',
    url: 'https://cloudcodetree.com/tutorials/<slug>/',
    siteName: 'CloudCodeTree', type: 'article',
    images: [{ url: '/tutorials/covers/<slug>.png', width: 1200, height: 675 }],
  },
  twitter: { card: 'summary_large_image', images: ['/tutorials/covers/<slug>.png'] },
};

<TutorialHero slug="<slug>" />

# <Subtitle>

**TL;DR: …**

<Callout tone="tip">  {/* verified type — link the companion repo + per-step diffs */}
**Code along, version by version.** Full project at
[github.com/cloudcodetree/tutorial-<slug>](https://github.com/cloudcodetree/tutorial-<slug>).
Each step is a git tag — `git checkout step-01`, then `step-02`, …
</Callout>

## What you'll be able to do after this
- …
```

For an **anchored** tutorial, drop the companion-repo callout and instead add a
`<Callout tone="warn">` noting where the code runs, then link the official
notebook/doc (verify the URL is live first).

> MDX gotcha: a bare `<` before a digit/space in prose (e.g. `<1%`) parses as JSX
> and fails the build. Write "under 1%" or `&lt;1%`. (Inside fenced code it's fine.)

### 3. Ship

`pnpm run build` (article styling, `<Callout>`/`<TutorialHero>`, sitemap entry,
and nav are already wired). Commit + push; the deploy publishes it.

## How it renders

- `app/tutorials/layout.tsx` wraps everything in the site chrome (`ClientLayout`).
- `app/tutorials/(article)/layout.tsx` styles the MDX (mobile-safe: code scrolls,
  long tokens wrap) and adds the "← Back to Tutorials" link. The author writes the
  `# H1`.
- Custom MDX components (`<Callout>`, `<TutorialHero>`) live in
  `app/components/mdx/` and are exposed via the root `mdx-components.tsx`.
  `<TutorialHero slug>` renders the cover image + a `series · Part N/M` eyebrow
  (reads the manifest); the author still writes the `# H1` (the subtitle) below it.
- `scripts/generate-tutorial-covers.mjs` builds the hero/OG covers (committed);
  `scripts/generate-feeds.mjs` auto-discovers tutorial slugs for the sitemap.
